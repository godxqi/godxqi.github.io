网络编程

本文为网络编程学习笔记

网络编程的本质

使用操作系统提供的接口函数，使应用程序具有收发网络数据的能力

网络编程核心概念

  • 协议
  • 地址
  • 端口号
  • 服务端：等待连接的设备
  • 客户端：发起连接的设备

网络字节序：网络字节顺序采用大端模式，所以在小端系统中需要做字节序转换

网络编程模式

CS模型:客户端/服务端编程模式

-服务端长期暴露于网络，并等待客户端连接
-客户端发起连接动作，并等待服务端回应
-特点：
- 服务端无法主动连接客户端
- 客户端只能按照预定义的方式连接服务端

客户端

编程模式

1.准备网络连接
2.连接远程设备 connect()
3.收发数据
4.关闭连接

客户端连接后向服务端发送请求(协议数据)

服务端

编程模式

1.准备网络连接
2.绑定端口 bind()
3.进入端口监听状态 listen()
4.等待连接
端口有连接:accept()拿到socket

服务端长时间运行(死循环)接收客户端请求

服务端核心工作

绑定，监听，接收
服务端的socket只用于接收连接，不进行实际通信
当接收到连接时，accept()函数返回与客户端通信的socket
服务端socket产生用于通信的客户端socket

socket

int socket(int domain,int type,int protocol);
在通信域domain中创建一个类型为type,使用协议protocol的套接字.而且返回一个描写叙述字,也就相当于打开了一个特殊的文件

本质

拿到一个特殊文件的描述符，通过修改参数可以让这个文件提供不同类型的通信功能（比如网络通信,本地进程间通信）

参数

  • domain：套接字中使用的协议族

    • PF_INET：IPv4互联网协议族
    • PF_INET6：IPv6互联网协议族
    • PF_LOCAL：本地进程间通信的协议族
    • PF_PACKET：底层数据收发协议族
    • PF_IPX：Novell专用网络通信协议(互联网分组交换协议)

      不同协议中的地址表现类型形式不同，网络编程地址必须和协议类型匹配
      AF_INET与PF_INET值相等,但根据用的位置不一样会使用AF_INET(Adrress)和PF_INET(Protocol)

  • type：套接字数据传输类型消息，用于指定协议类型

    • SOCK_STREAM：流式数据(TCP)
    • SOCK_UGREAM：报文式数据(UDP)

  • protocol：设备间通信使用的协议信息，用于指定协议族中符合类型的具体协议

    • domain和type几乎可以唯一确定一种协议，因此这个参数通常为0
    • 即:0代表domain和type指定后的默认协议
返回值

返回值0为正常,-1则设置errno表示出错

shutdown

int shutdown(int socket,int how)
断开与socket的连接

参数

SHUT_RD: 停止从套接字接收数据
SHUT_WR: 停止套接字传送数据
SHUT_RDWR: 全然停止

bind

int bind(int socket,const struct sockaddr *address,socklen_t address_len);
为建立的特殊文件socket建立本地捆绑(主机地址/端口号)

参数
  • address:服务端信息.通用接口,需要根据不同协议来指定具体实现,参考下面地址类型
  • address_len:对应地址的长度
地址类型struct sockaddr

用来防止目的主机的主机IP与端口信息
在需要使用struct sockaddr声明socket地址时,需要根据不同协议来指定地址类型.比如sockaddr_in(Ipv4),sockaddr_un(Local),sockaddr_in6(Ipv6)这些结构体,但代码层面将这些结构在作为形参使用时,还是需要加上(struct sockaddr)的强制转换

成员:

  • sin_family:
    • Ipv4:AP_INET
  • sa_data[14]:根据不同协议来规定,以Ipv4为例子
    • sin_port
    • struct in_addr sin_addr:一个Ipv4的地址
      • 如果不关心bind绑定的ip地址,可以使用INADDR_ANY
      • 如果想在本机上可以访问,使用本地回环地址INADDR_LOOPBACK(127.0.0.1)
      • 如果想在局域网上可以访问,使用本机的局域网地址
      • 广播地址:INADDR_BROADCAST(255.255.255.255)

struct in_addr
{
u_int32_t s_addr; //32位无符号
}

我们常用的IP地址比如178.16.145.222作为一个字符串,需要使用inet_addr()转换成网络地址
sin_port和sin_addr都必须是网络字节序（NBO），一般可视化的数字都是主机字节序（HBO）。

返回值

返回值0为正常,-1则设置errno表示出错

connect

int connect(int sock, struct sockaddr* address, socklen_t len)
连接指定地址的远程设备,用于client连接server

本质

完成一个有连接协议的连接过程

成员
  • sock:客户端本地描述符
  • addr:server端的地址信息,目的主机的主机IP与端口信息
  • len:server端的地址信息的地址长度
返回值

返回值0为正常,-1则设置errno表示出错

listen

int listen(int sock, int backlog)

本质

把socket的属性更改为被动连接,设置好之后立即返回
将这个socket加入一个未完成序列,参考listen与队列

成员
  • sock:服务端本地描述符,称为监听套接字.只要服务端在运行,这个socket应该一直存在
  • backlog:队列最大长度，每次服务几个客户端
返回值

返回值0为正常,-1则设置errno表示出错

accept

int accept(int socket,struct sockaddr *address,socklen_t *address_len);
listen调用后调用,从已完成序列的队首取出一项连接返回给进程,已完成队列为空则堵塞(无客户端connect行为)

成员
  • sock:服务端本地描述符,称为监听套接字.只要服务端在运行,这个socket应该一直存在
  • address:客户端信息,一般仅初始化
  • address_len:客户端地址的大小.调用前，我们将由addrlen所引用的整数值置为由address所指向的socket地址结构的长度
返回值

返回的为套接字,这个套接字代表那个已经用三次握手建立起来的那个tcp连接

send和write

ssize_t send(int socket,const void *buf,size_t length,int flags);
ssize_t write(int handle, void *buf, int count);
将buf的数据放入发送缓冲区中

成员
  • sockfd/handle:接收消息的套接字的文件描述符。
  • buf/buf:要发送的消息,指向包含要传输数据的缓冲区的指针
  • size/count:要发送的字节数
  • flags:调用方式标志位.一般为0.指阻塞式发送,发送不成功会一直阻塞,直到某个信号终端中止一系列标志
    • MSG_NOSIGHAL:表示当前连接被关闭时不会产生SIGNAL信号
    • MSG_DONTTWAIT:表示非阻塞发送

flags为0时,send与write同等

返回值

已经送出的字节数,失败返回-1,连接断开返回0
send()成功返回仅代表数据无误的发送到了网络驱动设备上,不代表另一端接收了数据

sendto

ssize_t sendto(int sockfd, const void *buf, size_t len, int flags,const struct sockaddr *dest_addr, socklen_t addrlen);
发送消息到一个套接字,一般用于UDP,用于已经建立的连接时(TCP)将后两个成员置NULL和0,与send同等

成员

  • sockfd:接收消息的套接字的文件描述符。
  • buf:要发送的消息,指向包含要传输数据的缓冲区的指针
  • len:要发送的字节数
  • flags:调用方式标志位.同
  • dest_addr:目的主机的主机IP与端口信息
  • addrlen: addr所指地址的长度
返回值

同send

recv和read

int recv( SOCKET sockfd, char *buf, int len, int flags);
将数据从接收缓冲区放入buf中

成员
  • sockfd:接收端套接字描述符
  • buf:要接收的消息,指向包含要传输数据的缓冲区的指针
  • len:要接收的字节数
  • flags:调用方式标志位.一般为0.指阻塞式发送,发送不成功会一直阻塞,直到某个信号终端中止一系列标志
返回值

已经接收的字节数,失败返回1,连接断开返回0
send()成功返回仅代表数据无误的发送到了网络驱动设备上,不代表另一端接收了数据

recvfrom

ssize_t recvfrom(int sockfd,void *buf,size_t len,unsigned int flags, struct sockaddr *from,socket_t *fromlen)
接收远程主机经指定的socket传来的数据,并把数据传到由参数buf指向的内存空间

成员
  • sockfd:接收消息的套接字的文件描述符。
  • buf:要接收的消息,指向包含要传输数据的缓冲区的指针
  • len:要接收的字节数
  • flags:调用方式标志位.同
  • dest_addr:目的主机的主机IP与端口信息
  • addrlen: addr所指地址的长度

close

int close(int fd);
关闭所打开的文件
成功关闭文件返回0，否则返回-1

IP地址相关函数

  • in_addr_t inet_addr( const char*strptr );将IP字符串转换为符合网络字节序的整数

    注意需要字符串的标志””

  • int inet_aton( const charcp, struct in_addrinp );将IP字符串转换为符合网络字节序的整数，成功返回1，失败返回0
  • char* inet_ntoa( struct in_addr in );将符合网络字节序的整数地址转换为字符串形式

字节序函数

  • uint16_t htons(uint16_t hostshort);将一个16位数从主机字节顺序转换成网络字节顺序。
  • uint32_t htonl(uint32_t hostlong);将一个32位数从主机字节顺序转换成网络字节顺序。
  • uint16_t ntohs(uint16_t netshort);将一个16位数从网络字节顺序转换成主机字节顺序。
  • uint32_t ntohl(uint32_t netlong);将一个32位数从网络字节顺序转换成主机字节顺序。

errno的使用

errno

对应着系统最后一次的错误代码
errno不同数值所代表的错误消息定义在errno.h中

strerror

strerror(error)
会将errno的数值对应的具体错误信息打印出来
包含在string.h中

listen与队列

对于一个调用listen()进行监听的套接字,操作系统会给这个套接字维护两个队列
![][listen队列]

RTT是未完成序列中任意一项在未完成队列中留存的时间,取决于客户端和服务端

未完成序列

当tcp三次握手中,第一次syn包给服务器的时候,服务器会在未完成序列中创建一个跟这个syn包对应的一项
这一项可以看成半连接,这个半连接会在accept调用后从LISTEN变为SYN_RCVD,同时给客户端返回第二次握手包[syn,ack]

已完成序列

三次握手完成时,连接变为ESTABLISHED状态.将这个连接放入已完成序列中
