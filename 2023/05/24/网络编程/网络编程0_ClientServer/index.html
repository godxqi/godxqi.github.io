网络编程

本文为网络编程学习笔记

网络编程的本质

使用操作系统提供的接口函数，使应用程序具有收发网络数据的能力

网络编程核心概念

  • 协议
  • 地址
  • 端口号
  • 服务端：等待连接的设备
  • 客户端：发起连接的设备

网络字节序：网络字节顺序采用大端模式，所以在小端系统中需要做字节序转换

网络编程模式

CS模型:客户端/服务端编程模式

-服务端长期暴露于网络，并等待客户端连接
-客户端发起连接动作，并等待服务端回应
-特点：
- 服务端无法主动连接客户端
- 客户端只能按照预定义的方式连接服务端

客户端

编程模式

1.准备网络连接
2.连接远程设备 connect()
3.收发数据
4.关闭连接

客户端连接后向服务端发送请求(协议数据)

服务端

编程模式

1.准备网络连接
2.绑定端口 bind()
3.进入端口监听状态 listen()
4.等待连接
端口有连接:accept()拿到socket

服务端长时间运行(死循环)接收客户端请求

服务端核心工作

绑定，监听，接收
-绑定
int bind(int sock, struct sockaddr* addr, socklen_t addrlen)
-监听
int listen(int sock, int backlog)
backlog:队列长度，每次服务几个客户端
-接收
int accept(int sock, struct sockaddr* addr, socklen_t* addrlen)
返回与客户端通信的socket
执行后会进入阻塞状态

服务端的socket只用于接收连接，不进行实际通信
当接收到连接时，accept()函数返回与客户端通信的socket
服务端socket产生用于通信的客户端socket

函数接口

include <sys/socket.h>
include <sys/types.h>
int socket(int domain, int type, int protocal);
创建套接字，为网络连接做准备
int connect(int sock, struct sockaddr* addr, socklen_t len)
连接指定地址的远程设备
ssize_t send(int fd, void* buf, size_t n, int flags)
发送数据到远程设备
ssize_t recv(int fd, void* buf, size_t n, int flags)
接收远程设备发回的数据
int close(int fd)
关闭连接，销毁套接字
htons()
本机字节序转换为网络字节序

socket函数

返回值:返回用于通信的资源标识符
本质功能:提供不同类型的通信功能（本地进程间通信）
