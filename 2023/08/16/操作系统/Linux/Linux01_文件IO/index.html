linux常用文件IO接口

open函数

int open(const char *pathname, int flags, mode_t mode);

参数

  • pathname：文件路径名（绝对路径）或者文件名
  • flags：文件的打开方式
    • O_WRONLY:只写打开;
    • O_RDONLY:只读打开;
    • O_RDWR:读写方式打开;
    • O_CREAT:文件不存在则创建;
    • O_APPEND:文件末尾追加;
    • O_TRUNC:清空文件,重新写入;
    • O_NONBLOCK:默认打开文件时, 默认为阻塞方式, 如果需要非阻塞式打开就需要这个flag(一般用于设备文件)

      可以使用|来进行复用, 如: O_WRONLY|O_CREAT

  • mode：文件的权限，如“0600”

    使用4个数字指定权限, 例如0666为可读可写不可执行, 一般在使用O_CREAT时使用

返回值

返回一个文件描述符，当文件成功打开时，返回一个非负整数，当文件打开失败时，返回-1

write函数：

ssize_t write(int fd, const void *buf, size_t count);

参数

  • fd:对应打开的文件描述符
  • buf:把文件内容读取到一块空间buf中
  • count:期望要读取的字节数

返回值

如果顺利write()会返回实际写入的字节数.
当有错误发生时则返回-1, 错误代码存入errno 中.

count和阻塞非阻塞结合起来考虑, 有可能在未读够count的数值时, 根据文件属性被阻塞住, 或者因为文件未非阻塞直接返回, 导致count<返回值
ssize_t read(int fd, void *buf, size_t count);

参数

  • fd:对应打开的文件描述符h3
  • buf:把文件内容读取到一块空间buf中
  • count:期望要读取的字节数

返回值

成功时，返回读取的字节数（零表示结束）
出现错误时，返回-1

文件操作的一般步骤

open -> read/write -> close
使用open打开一个文件时, linux内核在进程中建立了一个打开文件的数据结构, 记录下我们打开的这个文件

静态文件

存在与块设备的文件系统中

动态文件

内核在内存中申请一段内存, 并且将静态文件的内容从块设备中读取到内存中特定地址管理存放

read/write等操作文件时, 实际上是更改动态文件, 且动态文件与块设备中的静态文件是不同步的, 在close关闭动态文件后, close内部内核将根据内存中的动态文件的内容更新块设备中的静态文件
因为块设备ROM操作不灵活, 而内存RAM可以随机访问

文件描述符

实质为一个数字, 在一个进程中表示一个特定的含义, 用于区别一个程序打开的多个文件
作用域: 当前进程
范围: >=0

文件IO与标准IO

文件IO指open, close, write, read等API函数构成的一套用于读写文件的体系, 应用层直接调用后会将数据直接给到内核, 内核中的buff再根据硬盘的特性来选择好的时机去最终写入到硬盘中, 但文件IO效率并不是最高的
标准IO指fopen, fclose, fwite, fread等一系列C库函数, 他们在文件IO上加了一层封装, 主要是在应用层添加了一个缓冲区, 标准IO库会根据操作系统单次write的最佳count来选择好的时机给到内核, 内核中的buff再根据硬盘的特性来选择好的时机去最终写入到硬盘中

硬盘中的静态文件和inode(i节点)

静态文件平时存放在硬盘中
一个硬盘分为两个区域:一个是硬盘内容管理表项, 另一个是真正存储内容的区域.操作系统访问硬盘时, 会先读取硬盘内容管理表, 从中找到我们要访问的那个文件的扇区级别的信息, 再用这个信息去查询真正存储内容的区域, 最后得到我们用的文件.
操作系统最初拿到的信息是文件名，之后第一步就是去查询硬盘内容管理表,这个管理表中以文件为单位记录了各个文件的各种信息,每一个文件有一个信息列表(我们叫inode/i节点，其实质是一个结构体, 这个结构体有很多元素, 每个元素记录了这个文件的一些信息, 包括文件名、文件在硬盘上对应的扇区号、块号等)
