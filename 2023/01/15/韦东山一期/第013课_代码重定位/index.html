Nand与重定位

把程序烧到SDRAM、Nor上可直接运行，但CPU无法直接从Nand中取代码执行，那为什么还能使用Nand启动？

        1.一上电，片内内存SRAM会把Nand上的前4K复制过去

        2.CPU从0运行(0对应的是SRAM的地址)

                如果程序大于4K，前4K的代码需要把整个程序读出来放到SDRAM，重新确认程序的地址，即重定位

Nor与重定位

Nor可以像内存一样的读，但不能像内存一样的写

            比如执行

mov r0,#0     

LDR R1,[R0]            /* 有效：读 */

STR R1,[R0]            /* 无效：写 */

因为NorFlash相当于硬盘，其中包含着重要的数据，想写数据需要执行一些特殊的操作

在Nor上设置一个想改动的全局/静态变量，发现无法更改

                                           /* 全局/静态变量包含在bin中，烧在NorFlash上 ，无法直接修改 */

004节

链接脚本的改进

ldrb执行过程：CPU把ldrb这条命令发给内存控制器，内存控制器到SDRAM中读到四个字节的数据，然后跳出一个返回给CPU(SDRAM 32bit    Nor 16bit)

strb执行过程：CPU把地址和数据发给内存控制器，内存控制器把32位的数据发给SDRAM，同时发出数据屏蔽信号DQM，比如CPU只写一个字节，CPU就会发出三个屏蔽信号，屏蔽掉三个不需要写的字节(SDRAM 32bit    Nor 16bit)

ldrb    strb，以字节为单位访问Nor和SDRAM，访问Nor16次，访问SDRAM16次

ldr       str ,   以四个字节为单位访问Nor和SDRAM，访问Nor8次，访问SDRAM4次(因为Nor位宽为16位，读一次访问两次)

发现BSS段意外将数据段中的全局变量也清零了，进入反汇编看到

30000002不是4的整数倍，所以并不是4字节对齐的

str：无论存入地址为多少都是以4个字节为单位存入，也就是说无论存入地址是0x30000000或0x30000001或0x300000002或0x30000003，存入都是以0x30000000开始，在这个地方就将0x30000000也存入了0，而这个地址就是全局变量的地址。所以输出为0的原因是bss的开始地址是0x30000002，并不是向4取整的，而str是以4字节的方式存入r3的值，所以在清零bss段时将全局变量一起清0了。

ldr：读取以4个字节为单位循环读取。假设存入0xfe，当从0x30000001开始读取时，从低地址->高地址（00-00-00-fe）(0x30000001-0x30000002-0x30000003-0x30000000)，所以r2就成了0xfe000000；当从0x30000002读取时从低地址->高地址（00-00-fe-00）(0x30000002-0x30000003-0x30000000-0x30000001)，所以r2就成了0x00fe0000

那么只要修改链接脚本，让bss段向4取整就解决了这个问题

/* sdram.lds */

/* 将0x30000000地址与0x800地址链接 */

SECTIONS {

   .text   0  : { *(.text) }

   .rodata  : { *(.rodata) }

   .data 0x30000000 : AT(0x800)

    {

        data_load_addr = LOADADDR (.data);    /*返回()内段的绝对加载地址 */

   . = ALIGN(4);                  /* 以防意外 */

        data_start = . ;        /* data_star = 0x30000000 */

        *(.data)

        data_end = . ;          /* data_end - data_start = data段长度 */

    }

008程序启动分析：

bl sdram_init()

bl命令并不是跳到0x30000478这个地址，而是跳到当前pc+offset(offset：链接器计算的偏移地址)的位置，在反汇编中写出了这个值只是为了方便你分析代码，所以虽然程序指定了运行时地址，但即使放到0地址也可以运行，因为它跳转的时候是用的偏移地址，与运行时地址无关

重点：反汇编文件里，B/BL   0x3xxxxxx，这个值只是方便查看，不是跳到这个地址

bl main

重定位完毕，程序继续执行下一条指令bl main，而此时0x5c4和0x300005c4都有main函数，但这个地方使用的是bl命令，所用的pc依旧是当前的pc值，所以它仍然继续在Nor中运行

为什么要写位置无关码的程序？

因为为了让我们的程序在SDRAM里运行，我们使用了重定位，但这就要求我们必须做到在重定位之前的代码必须是位置无关码，这样才能正常运行。（连接脚本指定我们程序的运行地址为0x30000000，为什么我们的在nor flash上的代码从0地址开始运行也能工作？这就说明我们这部分代码必须是建立在位置无关码的基础上的，

            对于位置有关码来说：最终执行时的运行地址和编译链接时给定的链接地址必须相同，               否则一定会出错。

0地址处运行和0x30000000处运行达到同样效果，需要我们保证，在重定位之前，也就是复制操作没有完成之前的代码，必须是位置无关的）。

怎么写位置无关码的程序?(程序在任何地址都能运行，不需要把它放到运行时地址)

使用位置无关码！不能使用绝对地址！通过看反汇编来判断是否使用绝对地址

a.使用相对跳转命令B/BL

b.重定位之前不可使用绝对地址：不可访问全局变量/静态变量(因为重定位之前只有一个地址)

                           不可访问有初始值的数组，不是位置无关码(因为这些初始值放在rodata里,使用绝对地址来访问)

c.重定位后可以使用绝对地址，使用ldr pc, =xxx来跳转到Runtime Addr

    ldr r1, =_start         /* 源：第1条指令运行时的地址 */

    sub r2, r2, r1          /* 长度 */

    bl copy2sdram  /* src, dest, len    源， 目的， 长度 */

    /* 清除BSS段 */

    ldr r0, =__bss_start

    ldr r1, =_end

    bl clean_bss  /* startADDR, endADDR */

    ldr pc, =main  /* 绝对跳转, 跳到SDRAM */

/* Init.c */

void copy2sdram(volatile unsigned int *src, volatile unsigned int *dest, unsigned int len)  /* src, dest, len */

{

    unsigned int i = 0;

    while (i < len)

    {

        *dest++ = *src++;            /* 从源地址取数据存到目的地址*/

        i += 4;

    }

}

                                   /* start, end */

void clean_bss(volatile unsigned int *start, volatile unsigned int *end)  

{

    while (start <= end)

    {

        *start++ = 0;

    }

}

/* Init.c */

void copy2sdram(void)

{

    /* 要从lds文件中获得 __code_start, __bss_start

     * 然后从0地址把数据复制到__cod e_start

     */

    extern int __code_start, __bss_start;        /* 声明为外部变量 */

    volatile unsigned int *dest = (volatile unsigned int *)&__code_start;

    volatile unsigned int *end = (volatile unsigned int *)&__bss_start;

    volatile unsigned int *src = (volatile unsigned int *)0;

    while (dest < end)

    {

        *dest++ = *src++;

    }

}

void clean_bss(void)

{

    /* 要从lds文件中获得 __bss_start, _end

     */

    extern int _end, __bss_start;

    volatile unsigned int *start = (volatile unsigned int *)&__bss_start;

    volatile unsigned int *end = (volatile unsigned int *)&_end;

    while (start <= end)

    {

        *start++ = 0;

    }

}

C函数怎么使用lds文件中的变量abc（上方的__bss_start, _end等）?

a. 在C函数中声明改变量为extern类型, 类型并不重要，比如:

    extern int abc;

    

b. 使用时, 要取址, 比如:            但在汇编程序中可以直接使用

     int *p = &abc;  // p的值即为lds文件中abc的值

原因：编译程序时，有一个symbol table(符号表)

    

结论：1.C程序不保存lds文件中的变量

          2.借助Symbol table保存lds的变量，使用时加上”&”得到它的值

                对于g_i，使用&g_i得到Addr

                为了保持代码一致，对于lds的a1，使用&a1得到值
