简单的汇编指令

简单的汇编指令

（一）LDR：读内存。

-LDR R0,[R1]假设R1的值为X，读取地址X上的数据(4字节)，保存到R0中。即把R1数据读出来到R0中

-LDR R0，Label ；Label为程序标号，Label必须是当前指令的-4~4KB范围内；该指令是将Label地址中的值给r0(一般为机器码，若用了.word x给出的就是x的地址值)https://blog.csdn.net/qq1113231395/article/details/83650069

LDR r, label 和 LDR r, =label的区别：

LDR r, =label 会把label表示的值(label的地址)加载到寄存器中，而LDR r, label会把label当做地址，把label指向的地址中的值加载到寄存器中。（LDR r, =label用作大范围的地址读取伪指令）

譬如 label的值是 0x8000， LDR r, =label会将 0x8000加载到寄存器中，而LDR r, label则会将内存0x8000处的值加载到寄存器中。

（二）STR：写内存命令。

STR R0，[R1]，＃8；将R0中的字数据写入以R1为地址的存储器中，并将新地址R1＋8写入R1。

STR R0，[R1，＃8]；将R0中的字数据写入以R1＋8为地址的存储器中。

（三）B：跳转

（四）MOV： MOV R0,R1            //把R1的值赋给R0

MOV R0,#0x100    //R0=0X100

MOV R0,#0x12345678 //error，因为一条ARM指令中仅有32位，而MOV和R0占了一定字节，剩下的空间不足以表示任意值(0x12345678)，只能表示简单值(比如0x100，这种数被称为立即数)

（五）LDR R0,=0X12345678    //R0=0X12345678

此为伪指令，它会被拆分为几条真正的RAM指令

LDR r0, label  和 LDR r0, =label的区别：

LDR r0, label  会把label当做地址，把label指向的地址中的值加载到r0中，相当于指针操作.

LDR r0, =label 会把label表示的值加载到r0中，相当于直接赋值。

譬如 label的值是 0x8000， LDR r0, =label会将 0x8000加载到r0中，而LDR r0, label则会将内存0x8000处的值加载到r0中。

（六）add R0,R1,#4：R0=R1+4

sub R0,R1,#4：R0=R1-4

sub R0,R1,R2：R0=R1-R2

（七）BL：brarch and link

bl xxx：

  • 跳到XXX
  • 把返回地址(下一条指令的地址)保存到lr寄存器

（八）ldm：读内存，写入多个寄存器，高编号的寄存器存放在高地址 

stm：把多个寄存器的值写入内存，高编号的寄存器存放在高地址 

-ia：过后增加        -ib预先增加        -da过后减少        -db预先减少

（九）BIC―――――位清除指令

指令格式：BIC{cond}{S} Rd,Rn,operand2

BIC指令将Rn 的值与操作数operand2 的反码按位逻辑”与”，结果存放到目的寄存器Rd 中。

指令示例：

-BIC R0,R0,#0x0F ；将R0最低4位清零，其余位不变结果存放在R0中。

-bic r0, r0, #(1<<7)  /* 清除7位 */

（十）MRS：MRS{条件}     通用寄存器，程序状态寄存器（CPSR或SPSR）

MRS指令用于将程序状态寄存器的内容传送到通用寄存器中。

（十一）MSR：MSR{条件}     程序状态寄存器（CPSR戒SPSR）_<域>，操作数

MSR指令用亍将操作数的内容传送到程序状态寄存器的特定域中。

pfun main;                                       // 定义一个函数：void (*main)(void);
