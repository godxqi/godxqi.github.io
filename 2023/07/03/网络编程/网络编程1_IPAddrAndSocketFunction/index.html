网络编程

本文为网络编程学习笔记

socket

本质

socket()是一个多功能函数，通过修改参数来提供不同类型的通信功能（比如网络通信,本地进程间通信）

socket函数

int socket(int domain, int type, int protocol);

参数

domain：套接字中使用的协议族

  • PF_INET：IPv4互联网协议族
  • PF_INET6：IPv6互联网协议族
  • PF_LOCAL：本地进程间通信的协议族
  • PF_PACKET：底层数据收发协议族
  • PF_IPX：Novell专用网络通信协议(互联网分组交换协议)

    不同协议中的地址表现类型形式不同，网络编程地址必须和协议类型匹配

type：套接字数据传输类型消息，用于指定协议类型

  • SOCK_STREAM：流式数据(TCP)
  • SOCK_UGREAM：报文式数据(UDP)

protocol：设备间通信使用的协议信息，用于指定协议族中符合类型的具体协议

  • domain和type几乎可以唯一确定一种协议，因此这个参数通常为0
  • 即:0待变domain和type指定后的默认协议

socket()第一个参数必须与第二个参数的成员对应上,且第二个成员需要使用struct sockadd *强转(具体见地址类型)

端口号

两字节无符号整数
0-1023作为特定端口被预定义,在一些特殊系统中1024-2048也被预定义了

IP地址

4字节地址族

构成

  • 网络标识:标识网络主机(设备)所在的网络
  • 主机标识:标识网络主机(设备)的具体地址

分类

按照网络标识/主机标识的比例来分类

D类：不分配网络标识和主机标识，常用于多播
E类：不分配网络标识和主机标识，常用于实验

如何区分地址类型:子网掩码

4字节整型数
子网掩码与IP地址配合使用用于区分网络标识和主机标识

从子网掩码得到网络标识
IP地址:192.168.15.10 子网掩码:255.255.0.0
那么网络标识为192.168

从子网掩码得到可用子网数
设:子网掩码为M.N.P.Q，则子网可用IP地址n =(256 -M)(256-N)(256 - P)*(256 - Q)
子网掩码:255.255.255.248，则可用子网数为8个,也可以说子网有8个地址
实际上也就是2^3=8,即248中0的个数

从子网掩码与IP地址得到子网地址
IP地址:211.99.34.33,掩码255.255.255.248,两个地址相&即可得子网地址211.99.34.32,
广播地址是子网中最后一个IP地址,即广播地址为211.99.34.39
结合可用子网数,那么只有8-2=6个地址可以分配给子网中的设备(子网地址/广播地址除外)

简洁表示法
IP地址:211.99.34.33,掩码255.255.255.248,一般可以表示为211.99.34.33/29
29即为子网掩码中1的个数:255.255.255.248即11111111 11111111 11111111 11111000

特殊地址
  • 0.0.0.0 / 0 保留,常用于代表”缺省网络”
  • 127.0.0.0 / 8 回环地址,常用于本地软件回送测试
  • 255.255.255.255 / 32 广播地址
私有地址

不在公网(互联网)使用,只在内网(局域网)使用

  • 10.0.0.0 - 10.255.255.255 / 8
  • 172.16.0.0 - 172.31.255.255 / 16
  • 192.168.0.0 - 192.168.255.255 / 24

地址类型

因为不同通信中地址的指定不同,对于socket()函数来说,它的参数只能使用struct sockaddr类型
我们需要根据不同协议来指定地址类型:sockaddr_in(Ipv4),sockaddr_un(Local),sockaddr_in6(Ipv6)

在这些结构体中,第一个成员都是sa_family,常用Ipv4对应的参数是AP_INET.第二个成员sa_data[14]对应的值则根据不同协议来规定

Ipv4

struct in_addr
{
u_int32_t s_addr; //32位无符号
}
我们常用的IP地址比如178.16.145.222作为一个字符串,需要使用inet_addr()转换成网络地址

IP地址相关函数

#include <arpa/inet.h>

  • in_addr_t inet_addr( const char*strptr );将IP字符串转换为符合网络字节序的整数
  • int inet_aton( const charcp, struct in_addrinp );将IP字符串转换为符合网络字节序的整数，成功返回1，失败返回0
  • char* inet_ntoa( struct in_addr in );将符合网络字节序的整数地址转换为字符串形式
