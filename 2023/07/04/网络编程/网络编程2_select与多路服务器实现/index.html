Linux一切皆文件

Linux下的文件

  • 狭义:
    • 文件系统中物理意义上的文件(逻辑上关联的数据集合)
  • 广义
    • 设备,管道,内存….
    • Linux管理的一切对象

文件描述符(File Descriptor)

  • 文件描述符是一个非负整数值，本质是一个句柄
  • 一切对用户(程序员)透明的资源标识都可以看作句柄
  • 用户使用文件描述符(句柄)与内核交互
  • 内核通过文件描述符操作对应资源的数据结构

意义

  • 统一各种设备的操作方式(open,read,write,close)
  • 如:
    • IO设备(命令行,显示器)
    • 网络设备(网卡)
    • …..

所以对于socket,我们也可以使用这种方式操作

编程模式

int fd = open(“dev”, O_RDWR);
if(fd != -1)
{
char buf[32] = {0};
int len = read(fd, buf, sizeof(buf));
len = write(fd, buf, len);
close(fd);
}

事件相关函数

阻塞与非阻塞

  • 阻塞式函数:conncet()、accept()、recv()或recvfrom()
    • 函数调用后需要等待某个事件发生后才会返回
  • 非阻塞式函数:select()
    • 函数调用后能够及时返回(仅标记等待的事件)
    • 事件发生后以回调方式传递

轮询

  • 轮询指依序询问每一个相关设备是否需要服务的方式
  • 轮询可用于解决阻塞函数导致程序无法继续执行的问题

select()函数

select()用于监视指定的文件描述符是否产生事件,且通过轮询的方式检测目标事件(事件产生则标记发生变化)
int select(int maxfdp, fd_set* readfds, fd_set* writefds, fd_set* errorfds, struct timeval* timeout);

具体功能

  • 是否有对象需要接受数据
  • 是否有对象需要传输数据
  • 是否有对象发生了异常

参数

  • maxfdp:监视对象文件描述符的对象，一般就是所监视的最大文件描述符+1
  • readfds:指向fd_set结构的指针.readset中为1的文件描述符，将监视它们是否需要接受数据
  • writeset:writeset中为1的文件描述符，将监视它们是否需要传输数据
  • exceptset:exceptset中为1的文件描述符，将监视它们是否发生了异常
  • timeout:最大等待时间

struct fd_set
fd_set是一个文件描述符的集合.在Unix系统中,一切皆文件,自然socket也可以视为一个文件
fd_set可以通过一些宏人为来操作:

  • FD_ZERO(fd_set* fdset):清空集合/fd_set所指变量的所有位置0
  • FD_SET(int fd, fd_set* fdset):将fd加入fdset集合/将fdset所指变量的第fd位置1
  • FD_CLR(int fd, fd_set* fdset):将fd从set集合中清除/将fdset所指变量的第fd位置0
  • FD_ISSET(int fd, fd_set* fdset):在调用select()函数后，用FD_ISSET来检测fd是否在set集合中，当检测到fd在set中则返回真，否则返回假/第fd位为1返回真,否则返回假

struct timeval
用来代表时间值，有两个成员，一个是秒数，另一个毫秒数
struct timeval
{
long tv_sec; //秒
long tv_usec; //微秒
}

返回值

发生错误时返回-1，超时时返回0。如果发生监视的事件，返回相应的文件描述符
select()返回后,会将发生事件的位置都置起来,没有事件发生的位清空.
所以如果想重复监控,需要额外使用一个array保存放到监控集中的fd,用于在select返回后,在下一次select进入前,重新取得fd逐一加入(FD_ZERO最先),且需要扫描array来获得select的第一个参数

文件描述符的上限

监控的文件描述符个数取决于sizeof(fd_set)的值。我这边服务器上sizeof(fd_set)＝512，每bit表示一个文件描述符，则我服务器上支持的最大文件描述符是512*8=4096。据说可调，另有说虽然可调，但调整上限受于编译内核时的变量值

多路复用的服务端
  • 动态调整需要监视的文件按描述符
  • 当接收到客户端连接时，将客户端文件描述符加入监听变量(fd_set)中
  • 当发现客户端断开时，在监听变量(fd_set)中剔除客户端文件描述符
